The New CSS Height Units: lvh, svh, and dvh Explained

For years, developers have struggled with unpredictable viewport heights in mobile browsers. If you’ve ever seen a perfectly sized hero section suddenly crop or jump when the address bar hides, you’ve dealt with this problem firsthand. Thankfully, the CSS Working Group has addressed it with new height units: lvh, svh, and dvh.

The Problem with vh

The original vh unit (viewport height) always seemed simple: 1vh = 1% of the viewport’s height. But on mobile browsers, the viewport size changes when UI elements (like toolbars) appear or disappear. This leads to layout shifts when scrolling — not great for user experience.

Meet the New Units

  • lvh (Large Viewport Height) — The maximum possible viewport height, with all browser interfaces hidden.
  • svh (Small Viewport Height) — The minimum possible viewport height, when all overlays are visible.
  • dvh (Dynamic Viewport Height) — A real-time unit that updates as the browser UI expands or collapses (replaces traditional vh in many cases).

How They Work in Practice

.hero {
  height: 100dvh; /* Adapts dynamically */
  background: linear-gradient(to bottom, #2a2a72, #009ffd);
}

Try swapping in lvh or svh to see how your layout behaves when the viewport shrinks or expands. On mobile, these differences are immediately noticeable.

When to Use Which

  • For full-page sections: Use dvh to match the live viewport and prevent awkward cropping.
  • For fixed headers or footers: svh can ensure consistent layout even when browser UI is visible.
  • For immersive experiences (like PWAs): lvh gives you the full-screen real estate when browser chrome disappears.

Browser Support

As of early 2026, these units are widely supported across modern browsers, including Chrome, Safari, Firefox, and Edge. Always double-check Can I Use for the latest compatibility info.

The arrival of lvh, svh, and dvh marks a big win for responsive design. With these tools, developers finally have fine-grained control over viewport sizing — no more JavaScript hacks or layout jumps when scrolling.
